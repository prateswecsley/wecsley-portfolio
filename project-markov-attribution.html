Project Overview

As a Data Scientist at LexisNexis, I developed a sophisticated multi-touch attribution model using Markov Chains to optimize a $50M+ annual marketing budget. The project revolutionized how the company understood marketing channel effectiveness by moving beyond simplistic last-click attribution to capture the full customer journey.

The Challenge: LexisNexis was using last-click attribution, which credited 100% of conversion value to the final touchpoint. This severely undervalued awareness and consideration channels (Display, Social) while over-crediting direct traffic and branded search. Marketing teams couldn't confidently justify budget allocations.

Phase 1: Data Collection Strategy

üí° Methodological Decision: Multi-Source Integration Approach

Problem: Customer touchpoint data scattered across 8 disconnected systems (Google Analytics, Facebook Ads, Email platform, CRM, etc.)

Solution Chosen: Build centralized data lake in AWS S3 with automated ETL pipelines

Why This Approach:

  • ‚úì Enables cross-platform user ID matching (critical for journey reconstruction)
  • ‚úì Handles data at scale (5M+ touchpoints/month)
  • ‚úì Supports incremental updates without full reprocessing
  • ‚úì Decouples data extraction from modeling (failed API calls don't break pipeline)

üîÑ Data Collection Workflow

1
Google Analytics - Website Touchpoints
Data Extracted: User sessions, page views, traffic source, timestamps, conversion events Method: Google Analytics Data API v1 Beta with custom dimensions Why GA: Single source of truth for on-site behavior with user_id tracking Challenge Solved: Linked anonymous sessions to known users post-login using User-ID feature
2
Ad Platform APIs - Paid Channel Data
Platforms: Google Ads, Facebook Ads Manager, LinkedIn Campaign Manager Data Extracted: Click timestamps, campaign IDs, ad creative, gclid/fbclid parameters Why Separate Pull: Ad platforms have richer campaign metadata than GA UTM tags Integration Key: Matched click IDs (gclid, fbclid) to GA sessions for attribution
3
Email Marketing Platform - Email Touchpoints
Platform: Marketo Marketing Automation Data Extracted: Email sends, opens, clicks with lead_id and timestamp Why Critical: Email often mid-journey nurture; missed by web analytics Matching Strategy: Joined on email address to customer_id in CRM
4
CRM - Conversion Ground Truth
System: Salesforce CRM Data Extracted: Opportunity creation date, close date, revenue, customer_id Why Essential: Defines what constitutes a "conversion" for attribution Business Rule Applied: Only attributed opportunities created within 90 days of first touchpoint
5
Data Consolidation in AWS S3
Storage Format: Parquet files partitioned by date for efficient querying Orchestration: Apache Airflow DAGs running daily at 2 AM Why S3: Cost-effective storage for 18 months of historical data (2.3TB) Processing Layer: Databricks for Spark-based data transformations at scale

Phase 2: Data Cleaning & Journey Construction

üéØ Key Decision: Channel Taxonomy Design

Challenge: 50+ unique source/medium combinations from different platforms

Examples of inconsistency:

  • "google/cpc" vs "Google Paid Search" vs "google-ads" (all same channel)
  • "(direct)/(none)" vs "Direct Traffic" vs "direct-visit"

Solution: Created 8-category master taxonomy:

  1. Paid Search (Google Ads, Bing Ads)
  2. Organic Search (unpaid Google, Bing traffic)
  3. Display Advertising (banner ads, programmatic)
  4. Social Media (Facebook, LinkedIn, Twitter)
  5. Email Marketing (Marketo campaigns)
  6. Direct Traffic (type-in, bookmarks)
  7. Referral (partner sites, PR mentions)
  8. Affiliate Marketing (partner links)

Implementation: Rule-based classification with manual QA on 1,000 random journeys (98.7% accuracy)

üßπ Data Cleaning Workflow

1
Duplicate Session Removal
Problem: Users opening multiple tabs generated duplicate touchpoint records Detection Method: Grouped by (user_id, timestamp, channel) within 5-minute window Resolution: Kept earliest record, dropped duplicates Impact: Reduced dataset by 12.3% while preserving journey integrity
2
Journey Boundary Definition
Question: How long is a "customer journey"? Analysis Performed: Plotted time-between-touchpoints distribution Finding: 90% of touchpoints occur within 24 hours of previous one Decision: Split journeys when gap >24 hours (indicates new purchase intent)
3
Bot Traffic Filtering
Problem: Automated bots creating fake journeys Indicators: >50 page views/session, <1 second session duration, suspicious user-agents Filtration: Removed sessions matching 2+ bot characteristics Result: Eliminated 3.8% of sessions (significantly skewing conversion rates)
4
Journey Sequence Construction
Goal: Transform touchpoint table into customer journey sequences Method: Ordered by timestamp, concatenated channels per user_id Format: "Paid Search ‚Üí Organic Search ‚Üí Email ‚Üí Direct ‚Üí CONVERSION" Output: 800,000+ unique customer journeys with conversion flag

Phase 4: Markov Chain Model Development

üß† Why Markov Chains for Attribution?

Alternative Models Considered:

  • Linear Attribution (equal credit to all touchpoints) - Too simplistic
  • Time-Decay (more credit to recent touchpoints) - Arbitrary decay function
  • Position-Based (40-20-40: first-middle-last) - Fixed weights don't reflect reality

Why Markov Chains Won:

  • ‚úì Data-driven: Learns channel value from actual customer behavior
  • ‚úì Probabilistic: Captures uncertainty in attribution
  • ‚úì Removal Effect: Quantifies "what if we removed this channel?"
  • ‚úì Accounts for sequence: Order of touchpoints matters

‚öôÔ∏è Markov Chain Implementation Workflow

1
State Space Definition
States: Start ‚Üí Paid Search ‚Üí Organic Search ‚Üí Display ‚Üí Social ‚Üí Email ‚Üí Direct ‚Üí Referral ‚Üí Affiliate ‚Üí Conversion ‚Üí Null Start State: Entry point for all customer journeys Conversion State: Terminal state (journey complete) Null State: Terminal state for abandoned journeys (no conversion)
2
Transition Matrix Calculation
Method: Counted all transitions between states across 800K journeys Formula: P(State B | State A) = Count(A‚ÜíB) / Count(A) Example: P(Conversion | Email) = 12% (12% of journeys touching Email convert) Example: P(Organic Search | Paid Search) = 35% (common research path) Output: 10x10 transition probability matrix
3
Removal Effect Calculation
Question: What happens to conversion rate if we remove Channel X? Method: For each channel, recalculate transition matrix with that channel removed Example: Remove Email ‚Üí conversion probability drops from 3.8% to 3.1% Interpretation: Email is responsible for 18.4% of conversions (removal effect) Application: Removal effects sum to 100%, giving attribution weights
4
Channel Attribution Scoring
Formula: Channel Contribution = (Base Conversion Rate - Removed Conversion Rate) / Base Rate Total Conversions: 12,450 in 90-day period Total Revenue: $47.2M attributed Distribution: Email 23%, Paid Search 19%, Organic 18%, Display 15%, Social 12%, Direct 8%, Others 5%
